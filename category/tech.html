pelicanのデフォルト設定がREADME.mdを消す

他のバージョンだとわからないけど、現地点の最新のバージョンだと デフォルトで、make github したときに publishconf.pyを読んで、README.mdがいらない子だと判断されて消されてしまいます。

  • なので、publishconf.py の設定でこれが必要
DELETE_OUTPUT_DIRECTORY = False

むしゃくしゃして、githubのweb上からREADME.mdを作成して コンフリクト して、寝る前にやったから思い出せなくて、原因が思い出せない。(pelicanユーザなら通る道なのかな)

README.mdだけ消さない設定があるかもしれないけど、 メモ代わりブログでがんばりたくない。

tmux用のzshrc

完全に個人用設定の .zshrc

# ansibleで遊ぶ用
export CLOUDSDK_PYTHON_SITEPACKAGES=1
export ANSIBLE_HOST_KEY_CHECKING=False

# zsh起動時に強制的にvirtualenvに入る
if [ -n "$VIRTUAL_ENV" ]; then
  . "$VIRTUAL_ENV/bin/activate"
else
  workon py27
fi

# tmuxでsshで他サーバのログイン時にウィンドウ名をホスト名にする
ssh_tmux() {
    ssh_cmd="ssh $@"
    tmux new-window -n "$*" "$ssh_cmd"
}

if [ $TERM = "screen" ] ; then
    tmux lsw
    if [ $? -eq 0 ] ; then
        alias ssh=ssh_tmux
    fi
fi
